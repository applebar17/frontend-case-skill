# Role Levels

Use these levels to tune the case scope. If the requested role level conflicts with the CV signals, follow the requested role level and mention the mismatch as an assumption.

## Junior

Focus on implementation basics, readable components, simple state, semantic HTML, and clear handling of loading/error/empty states.

Expected case traits:

- Narrow feature scope.
- Explicit requirements.
- Minimal architecture decisions.
- One or two key interactions.
- Basic tests or testing plan.

## Mid

Focus on independent delivery of a complete UI workflow with sensible component boundaries and pragmatic testing.

Expected case traits:

- Moderate feature scope.
- Several UI states.
- Typed data model.
- Basic API/mock integration.
- Accessibility and responsive expectations.

## Senior

Focus on architecture, maintainability, tradeoffs, performance, accessibility, and test strategy.

Expected case traits:

- Realistic product scenario.
- Non-trivial state and data handling.
- Clear component/API boundaries.
- Explicit tradeoff discussion.
- Testing and accessibility are first-class.

## Lead

Focus on system design, team-scale maintainability, design systems, cross-functional constraints, and technical leadership.

Expected case traits:

- Broader workflow or component system.
- Design system or platform constraints.
- Migration, reuse, or governance tradeoffs.
- Review guidance and mentoring prompts.
- Clear expectations around technical communication.

## Architect

Focus on frontend platform strategy, integration boundaries, performance budgets, security constraints, operability, and long-term evolution.

Expected case traits:

- Architecture proposal plus focused implementation slice.
- Platform constraints and decision records.
- Performance, reliability, and security considerations.
- Multiple viable options with tradeoffs.
- Strong emphasis on communication and governance.
