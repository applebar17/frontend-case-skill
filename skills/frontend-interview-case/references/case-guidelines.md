# Case Guidelines

## Case Shape

Create one practical frontend assignment that can be completed in 90 to 180 minutes unless the user asks for a different format. The case should test how the candidate thinks, structures UI work, communicates tradeoffs, and handles real delivery constraints.

Include these sections:

1. Title
2. Target level
3. Estimated duration
4. Scenario
5. Candidate task
6. Requirements
7. Constraints
8. Deliverables
9. Evaluation rubric
10. Interviewer notes
11. Follow-up discussion prompts

## Good Case Patterns

- Build or refactor a focused UI workflow.
- Integrate with a small mocked API or provided JSON payload.
- Handle loading, empty, error, and success states.
- Include accessibility expectations.
- Include at least one testing expectation.
- Include at least one maintainability or architecture tradeoff.
- Ask for explanation of decisions, not only code.

## Avoid

- Trivia-only quizzes.
- Tasks that require obscure framework internals.
- Assignments that depend on unpaid business work.
- Assignments copied from the candidate's exact CV projects.
- Requirements that require protected personal information or demographic inference.
- Fully open-ended "build anything" prompts.

## Tailoring Guidance

Use the CV to tune complexity, not to reproduce private work history.

- If the CV shows strong React and TypeScript experience, use React and TypeScript unless the user asks otherwise.
- If the CV shows design system or component library work, include component API design, theming, or reuse constraints.
- If the CV shows performance work, include measurable rendering, data loading, or bundle tradeoffs.
- If the CV shows accessibility work, include keyboard navigation, semantic markup, focus management, and screen reader expectations.
- If the CV shows leadership, include architecture decisions, mentoring notes, review prompts, and stakeholder tradeoffs.
- If the CV shows limited frontend experience, keep the task narrow and reduce framework assumptions.

## Default Technical Constraints

Use these defaults unless the user or repo context says otherwise:

- TypeScript preferred.
- React preferred for React-heavy CVs.
- No backend required; use mocked data.
- Include responsive behavior for mobile and desktop.
- Include keyboard accessible interactions.
- Include tests for the most important behavior.
- Include a short README or notes section explaining choices.

## Candidate Privacy

Use neutral phrasing. Do not include employer names, exact project names, dates of employment, education details, personal contact data, or location data from the CV in the generated case unless the user explicitly requires it for internal interviewer notes.
